# 贡献指南

感谢你改进 memEcho Skills。仓库优先接受可验证、证据边界清楚且不扩大心理推断风险的改动。

## 开发流程

1. Fork 仓库并从 `main` 创建短分支。
2. 修改 Skill 时保持 `SKILL.md` 简洁，把详细合同或策略放在 `references/`。
3. 新增行为时补充一个最小示例或自动化测试。
4. 运行 `python -m unittest discover -s tests -v`。
5. 提交 Pull Request，说明用户问题、行为变化、验证方式和隐私影响。

## 设计原则

- 只陈述证据支持的最弱结论；区分观察、计算和解释。
- 不把 VAD 当作心理诊断、人格结论、欺骗检测或真实内心状态。
- 文本模式不推断声学特征。
- “事实主张”表示可外部核验的表达方式，不代表已经核实为真。
- 多人自我回声和陪练必须由用户确认“我”的身份。
- 建议动作保持 `proposed`，不得冒充来源中的明确承诺。
- 长期记忆与外部动作必须单独授权。

## 兼容性

合同的 `1.x` 版本只接受向后兼容的增量字段。改变必填字段含义或删除字段需要新的主版本。消费者应忽略未知字段。
