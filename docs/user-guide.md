# memEcho 用户指南

## 1. 输入什么

memEcho 接受文本、带说话人/时间戳的逐字稿，以及在平台具备相应能力时的音频或视频。

建议输入至少包含：

- 对话内容或文件；
- 对话场景，例如会议、访谈、沟通复盘或独白；
- 要重点分析的人；
- 多人场景中谁代表“我”；
- 需要的模块：纪要、内容解析、VAD、自我回声、陪练；
- 是否允许长期记忆，默认为不允许。

单人文本中，唯一说话人自动视为“我”。多人文本中，memEcho 不会根据姓名、代词、账号、音色或发言量猜测“我是谁”。

## 2. 五种分析模块

### 内容纪要

适合会议和决策型谈话，输出摘要、共识、未解决分歧、关注重点、明确动作和建议。

- 对话中明确承诺的动作：`origin: discussed`、`status: confirmed`。
- 分析者补充的建议：`origin: suggested`、`status: proposed`。

### 内容解析

按参与者拆分三类表达：

- 事实主张：被表达为可以外部核验的内容，但分类本身不核实真假；
- 观点：判断、推论、偏好、预测或建议；
- 态度：对人、方案、事件或结果的支持、抵触、不确定、担忧或承诺。

多人对话会进一步总结某句话之后出现的对齐、抵触、澄清、修复、沉默或转题，不把时间先后直接写成因果。

### VAD 走向

VAD 分别表示：

- Valence：表达的正负向；
- Arousal：表达的激活或紧张程度；
- Dominance：表达的掌控感或退让程度。

只有文本时，输出是语言表达层面的推断，`acoustic_weight` 必须为 `0`。只有平台确实读取并计算原始音频时，才可以使用声学证据。

### 自我回声

从用户确认的“我”出发，逐段回答：

1. 我使用了什么表达；
2. 对方随后出现了什么可观察反应；
3. 之后的 VAD 表达怎样变化；
4. 哪些部分可能有效，哪些部分与摩擦相伴；
5. 还有什么替代解释；
6. 如何换成更具体、低压力的话术。

### 陪练模式

陪练每次只呈现一个场景和一个目标，等待用户真实作答后再评分。评分维度包括清晰度、倾听/承接、情绪负荷、事实与判断分离、行动性。未回答的练习保持 `awaiting_user`，不会虚构回答或提前打分。

## 3. 推荐提问模板

### 单人独白

```text
memecho analyze
这是我的单人独白，请分析内容摘要、事实/观点/态度和语言侧 VAD 走向。
重点指出自我修正、压力话术与可执行动作。不要写入长期记忆。
[粘贴文本]
```

### 多人会议

```text
memecho analyze --self 阿青 --target 阿青,小林
场景是产品评审。请给出纪要、共识、分歧、明确动作与建议；
分别分析两人的事实、观点、态度，并从阿青的视角做自我回声。
[粘贴逐字稿或附文件]
```

### 只看某人的 VAD

```text
memecho vad --target 小林
请按稳定时间段分析小林的语言侧 VAD 走向，每个转折给出证据、置信度与替代解释。
[粘贴逐字稿]
```

### 互动陪练

```text
memecho coach --self 阿青
从本次对话里选一个卡点开始陪练。每轮只问一个问题，等我回答后再评分。
[粘贴逐字稿]
```

## 4. 如何读报告

- `分析范围`：先确认分析的是单次对话、哪些人和哪些信号。
- `confidence`：表示该解释被证据支持的程度，不是情绪强度。
- `alternative`：提醒同一现象可能有其他解释，例如编辑、转写错误或上下文缺失。
- `evidence_refs`：将结论连接到时间戳或稳定段落编号。
- `text_only`：表示报告没有使用声学证据。
- `memory.written: false`：表示没有写入长期记忆。

### Markdown 与 ASCII 图

1.0.2 默认在 Markdown 的数值摘要中加入纯 ASCII 图。`*` 表示当前 VAD 值，`|` 表示零点，`#` 表示置信度已填充部分。数字仍是权威值，图形只用于快速扫描，不增加分析精度。

### HTML 报告

需要展示、打印或本地归档时，可要求 `--format html`。HTML 报告包含纪要卡片、明确动作/建议状态、参与者内容解析、VAD 仪表、洞察、证据索引和不确定性。文件完全自包含，可离线打开；所有来自对话的文本都必须经过 HTML 转义。

## 5. 能力边界

memEcho 不用于心理诊断、人格评估、测谎、隐藏意图判断，也不应作为医疗、就业、教育、保险、法律或安全决策的自动依据。对话参与者应了解录音、上传和分析的范围。
